# Task 001: 프로젝트 구조 및 라우팅 설정

## 개요

Next.js App Router 기반으로 전체 라우트 구조를 생성하고, 인보이스 관련 페이지들의 빈 껍데기를 구성합니다.
이 작업은 구조 우선 접근법에 따라 실제 기능 구현 전에 애플리케이션의 전체 골격을 완성합니다.

## 관련 기능

- 기초 설정 (모든 기능의 전제 조건)

## 관련 파일

- `app/(main)/invoices/page.tsx` - 인보이스 목록 페이지
- `app/(main)/invoices/[id]/page.tsx` - 인보이스 상세 페이지 (동적 라우트)
- `components/nav-menu.tsx` - 네비게이션 메뉴 (인보이스 링크 추가)

## 수락 기준

- [ ] `/invoices` 경로로 인보이스 목록 페이지 접근 가능
- [ ] `/invoices/[id]` 경로로 인보이스 상세 페이지 접근 가능
- [ ] 네비게이션 메뉴에 인보이스 목록 링크 추가됨
- [ ] 모든 페이지에서 공통 레이아웃 (헤더/푸터) 적용됨
- [ ] 타입 에러 없이 빌드 성공

## 구현 단계

### 단계 1: 인보이스 목록 페이지 확인

- [ ] 1.1 `app/(main)/invoices/page.tsx` 파일 존재 확인
- [ ] 1.2 기본 페이지 컴포넌트 구조 확인/수정
- [ ] 1.3 "인보이스 목록" 제목 표시

### 단계 2: 인보이스 상세 페이지 생성

- [ ] 2.1 `app/(main)/invoices/[id]` 디렉토리 생성
- [ ] 2.2 `page.tsx` 동적 라우트 페이지 생성
- [ ] 2.3 URL 파라미터 (id) 표시하는 기본 구조

### 단계 3: 네비게이션 업데이트

- [ ] 3.1 `components/nav-menu.tsx`에 인보이스 링크 추가
- [ ] 3.2 현재 경로 활성 상태 스타일 확인

### 단계 4: 검증

- [ ] 4.1 `npm run build` 성공 확인
- [ ] 4.2 `npm run dev`로 모든 라우트 접근 테스트
- [ ] 4.3 타입 에러 없음 확인 (`npm run type-check`)
